# Examples

This directory contains example files to help you get started with the EcoFlow Cloud integration.

---

## dashboard_ecoflow_v1.0.yaml

A complete Home Assistant dashboard for the EcoFlow Delta 3 1500, covering all integration entities:

- Battery status & health
- AC input / output
- Solar input
- DC outputs (12V, 24V, car port)
- USB outputs (USB-A, USB-A QC, USB-C, wireless)
- Settings & controls
- Battery detail (BMS)
- Lifetime statistics

### How to use

1. In Home Assistant, go to **Settings → Dashboards**
2. Click **Add Dashboard** → give it a name (e.g. `EcoFlow`)
3. Open the new dashboard → click **⋮ (three dots)** → **Raw configuration editor**
4. Paste the contents of `dashboard_ecoflow_v1.0.yaml`
5. Save

> **Note:** The dashboard uses entity IDs generated by the integration for a device named `EcoFlow Delta 3 1500`.
> If your device has a different name, the entity IDs will differ (e.g. `sensor.ecoflow_delta_3_plus_battery_level`).
> Adjust the entity IDs in the YAML accordingly.

## ecoflow_optimal_charging_v1.3.yaml

A ready-to-use Home Assistant automation that combines two charging modes in one:

**Solar charging (07:00–23:00):** adjusts the Delta 3 charge power every minute to consume available solar surplus. Targets grid ≈ 0W. Pauses automatically when surplus drops below 200W.

**Emergency charging (always active):** starts automatically when SoC drops below 10%, charges at minimum 800W (scales up with surplus), stops at 30% SoC. Takes priority over solar charging.

### Constants to review before activating

| Constant | Default | Description |
|---|---|---|
| `MIN_CHARGE_POWER` | 200 W | Minimum charge power in solar mode |
| `MAX_CHARGE_POWER` | 1500 W | Maximum charge power |
| `SOC_MAX` | 90 % | Stop solar charging above this SoC |
| `EMRG_MIN` | 800 W | Minimum charge power in emergency mode |
| `SOC_EMRG_START` | 10 % | Start emergency charging below this SoC |
| `SOC_EMRG_TARGET` | 30 % | Stop emergency charging above this SoC |

### How to use

1. Copy `ecoflow_optimal_charging_v1.3.yaml` to your Home Assistant config folder
2. Check and update entity IDs at the top of the file to match your installation:
   - `sensor.power_usage` — replace with your P1/grid power sensor (negative = export)
   - `sensor.ecoflow_delta_3_1500_battery_level` — adjust if your device name differs
   - `number.ecoflow_delta_3_1500_ac_charging_speed` — idem
   - `switch.ecoflow_delta_3_1500_ac_charging_230v` — idem
3. In Home Assistant, go to **Settings → Automations** → **⋮ → Import automation**
4. Test with hardware actions set to `enabled: false` (default) — check the trace
5. Set all three hardware actions to `enabled: true` when ready

> **Tip:** Also set the **Max Charge Level** number entity to 90% in Home Assistant. This ensures the device itself stops charging at 90% as a hardware-level backup, independent of the automation.

## test_credentials.py

A command-line tool to verify your EcoFlow credentials and MQTT connection before adding the integration to Home Assistant. Useful for troubleshooting login or connectivity issues.

### What it does

- Logs in using your EcoFlow email and password (App Login mode)
- Retrieves your MQTT credentials from the EcoFlow cloud
- Connects to the MQTT broker and listens for device data for 30 seconds
- Prints all received MQTT keys and values to the console

### How to use

```bash
pip install requests paho-mqtt
python3 examples/test_credentials.py
```

Enter your email and password when prompted. The script will print your `userId`, MQTT credentials, and any incoming device data.

> **Note:** This script uses the same App Login flow as the integration itself.
> If credentials work here, they will work in Home Assistant.
